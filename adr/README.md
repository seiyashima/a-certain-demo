# ADR

ここには、今回の設計・開発で決まったことを記録する。

## 使い方

- 1 件 1 ファイルで記録する
- 迷った案ではなく、最終的に採用した判断を書く
- 変更理由、代替案、影響範囲を残す

## 一覧

- `0001-technical-selection.md`: エンタープライズ検索基盤の技術選定
- `0002-authorization-model.md`: 認可モデルとして RBAC + ABAC を採用

## テンプレート

```md
# 0001: タイトル

## Context

## Decision

## Consequences

## Notes
```
