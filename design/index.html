a-certain-demo 設計ドキュメント

Gemini Enterprise から呼び出される Cloud Run アプリとしての設計情報を、段階的に蓄積するための入口ページです。

ドキュメント一覧

  • ADR / 決定記録 : 今回の設計・開発で決まったことの記録
  • Cloud Run 求められる機能一覧 : Cloud Run アプリに必要な機能の整理
  • 連携アーキテクチャ設計 : 認証、ACL、監査、Connector 分離
  • Overall アーキテクチャ図 (SVG) : ユーザアクセス、SSO、Gemini Enterprise、Data Store の取得フロー
  • Overall アーキテクチャ図 (drawio) : Overall アーキテクチャ図の編集元ファイル
  • Connector アーキテクチャ図 (SVG) : Data Store への Fetch / Transform / Sync 詳細
  • Connector アーキテクチャ図 (drawio) : Connector アーキテクチャ図の編集元ファイル
  • Gemini Enterprise / Okta 認証認可シーケンス図 (Rendered Mermaid) : 認証認可部のみをレンダリングした表示ページ
  • Gemini Enterprise / Okta 認証認可シーケンス図 (Mermaid Source) : 編集用の Mermaid ソース
  • Gemini Enterprise ユーザ検索シーケンス図 (Rendered Mermaid) : 検索実行時のフローをレンダリングした表示ページ
  • Gemini Enterprise ユーザ検索シーケンス図 (Mermaid Source) : 編集用の Mermaid ソース
  • Agent Search 認可判定 詳細設計 : 一次判定（Agent Search）と高機密区分の二次判定（Cloud Run）の設計
  • SharePoint / Entra ID ログインシーケンス 詳細設計 : Okta フェデレーションの関与点と Entra ID のトークン発行責務を整理
  • システム概要設計 : 目的、構成、API、運用確認方針
更新ルール
  • 仕様変更時は該当 HTML を更新し、コミットに含める
  • 運用手順は README と重複させず、設計判断と背景を中心に書く
  • 新規ドキュメントはこのページからリンクする
  • 決まったことは ADR に残し、設計本文と分けて管理する
  • 機密要件は design/private に分離し、公開一覧には載せない
