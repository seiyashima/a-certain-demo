Detailed Design

SharePoint / Entra ID ログインシーケンス

ユーザが SharePoint にサインインする際の認証経路を示す。Okta は Entra ID のフェデレーション先として関与するが、 Graph 向けトークンの発行主体は Entra ID である。

1. シーケンス図

sequenceDiagram autonumber participant User as ユーザ/ブラウザ participant SharePoint as SharePoint Online participant Entra as Microsoft Entra ID participant Okta as Okta SSO User->>SharePoint: SharePoint へアクセス SharePoint->>Entra: 認証要求（未認証） Entra->>Entra: 対象ドメインのフェデレーション判定 Entra-->>User: Okta ログインへリダイレクト User->>Okta: ログイン（MFA 含む） Okta-->>User: 認証結果を返却 User->>Entra: 認証結果を提示 Entra->>Entra: トークン発行（M365/SharePoint 用） Entra-->>User: セッション/トークン返却 User->>SharePoint: トークン付きで再アクセス SharePoint-->>User: コンテンツ表示 Note over Okta,Entra: Okta はユーザ認証を担当。最終的な M365 向けトークン発行は Entra ID が担当

2. Okta はどう関与するか

観点 役割
ユーザログイン Okta が本人認証/MFA を実施する（フェデレーション連携）
SharePoint/Graph 向けトークン発行 Entra ID が発行主体。Okta 発行トークンをそのまま Graph に渡す設計ではない
Cloud Run クローリング時 ユーザログインと分離。Cloud Run は Entra ID（WIF 前提）でアプリ認証し Graph を呼び出す
