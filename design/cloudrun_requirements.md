# Cloud Run の求められる機能一覧

Gemini Enterprise Agent Platform から呼ばれる Cloud Run アプリを設計するための、機能要件のたたき台。

## 1. ユーザー向け機能

- 検索要求を受け付ける API を提供する
- 検索結果を返す前に ACL を判定する
- 検索結果を見やすい形に整形して返す
- 疎通確認のための UI を提供する
- UI から health / runtime / chat の確認ができる

## 2. データ取り込み機能

- 外部 SaaS から文書やメタデータを同期する
- 同期したデータを検索用に正規化する
- 再インデックスを実行できる
- 差分同期と全量同期を切り替えられる

## 3. 権限・認可機能

- Okta SSO を起点とした認証を扱う
- 元システムの ACL を保持して検索に反映する
- 文書単位またはフォルダ単位の閲覧制御を適用する
- 権限不足の要求は検索結果に含めない

## 4. 運用機能

- ヘルスチェックを返す
- 実行時設定やバージョンを返す runtime API を提供する
- リクエスト ID を使ってログを追跡できる
- エラー時に原因を追いやすい監査ログを残す
- デプロイ後にすぐ動作確認できる

## 5. 外部接続機能

- ServiceNow に接続できる
- Workday に接続できる
- Compliance System に接続できる
- SharePoint に接続できる
- Confluence に接続できる

## 6. 非機能要件

- Cloud Run 上でステートレスに動作する
- 自動スケールに追従できる
- 短時間で起動し、スモールスタートに耐える
- 失敗してもリトライ可能な設計にする
- 機密情報をコードやリポジトリに置かない

## 7. セキュリティ要件

- 長期資格情報を持たない
- WIF を使って外部サービスへ接続する
- 秘密情報は Secret Manager などの外部管理に寄せる
- 監査に必要な情報だけを最小限残す

## 8. 監視・品質確認

- Health / runtime / chat の疎通確認を実施できる
- 主要処理の成功・失敗を観測できる
- 処理時間を記録できる
- デプロイ後の回帰を簡単に検出できる
